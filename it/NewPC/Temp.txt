Windows — user-scope permissions for /end-session (no prompts)
================================================================

THE RIGHT FILE
--------------
Permission rules go in:   C:\Users\<UserName>\.claude\settings.json
                          (the .claude FOLDER, then settings.json)

NOT in C:\Users\<UserName>\.claude.json - that file holds MCP server
registrations and app state only. Leave it alone.

Note: "Yes, always allow" answers at prompts are saved to
<launch folder>\.claude\settings.local.json - they only help that one
folder. The user-scope file below works from every launch folder.


WHAT TO DO
----------
1. Open C:\Users\<UserName>\.claude\settings.json (create it if missing).

2. If the file already exists, MERGE the entries below into the existing
   "allow" array - do not replace the file. If it already has rules like
   Read(/docs/terminai/**) or Read(\\irwinnas\...) they are broken
   (wrong path form) and can be replaced by these.

3. If the file did not exist, paste this as the entire content:

{
  "permissions": {
    "allow": [
      "Read(//irwinnas/MyDocs/terminai/**)",
      "Write(//irwinnas/MyDocs/terminai/**)",
      "Edit(//irwinnas/MyDocs/terminai/**)",
      "Read(//I:/terminai/**)",
      "Write(//I:/terminai/**)",
      "Edit(//I:/terminai/**)",
      "Read(I:/terminai/**)",
      "Write(I:/terminai/**)",
      "Edit(I:/terminai/**)",
      "Bash(git status:*)",
      "Bash(git log:*)",
      "Bash(git diff:*)",
      "Bash(git add:*)",
      "Bash(git commit:*)",
      "Bash(git push:*)",
      "Bash(git pull:*)",
      "Bash(git check-ignore:*)"
    ]
  }
}

4. Restart Claude Code (settings load at session start), then re-test
   /end-session.


NOTES
-----
- Path rules always use FORWARD slashes, even on Windows.
- A leading double slash means an absolute path; the UNC share
  \\irwinnas\MyDocs\terminai becomes //irwinnas/MyDocs/terminai.
- The I:/ drive-letter rules are included in two forms (//I:/ and I:/)
  because the repo may be reached either via the mapped drive or the UNC
  path depending on how the session is launched - unused variants are
  harmless. If prompts still appear, note the exact path shown in the
  prompt (UNC vs I:) and match a rule to it.
- The "Unrecognized redirect shape" git commit prompt is fixed separately:
  the /end-session command no longer uses a heredoc for the commit
  message (pull the latest repo on the Windows machine to get the
  updated command).
- The .git\index.lock warning is a built-in safety check - it only
  appeared because an earlier failed commit left a stale lock. Do not
  add an allow rule for .git paths.
