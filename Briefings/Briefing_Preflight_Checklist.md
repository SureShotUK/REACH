# Briefing Pre-Flight Checklist

*Run every briefing through this checklist before it is called finished. A "No" on any starred (★) item means the briefing is not done.*

---

## 1. House Style (per master CLAUDE.md Briefing Format Standards)

- [ ] ★ Portland Long logo is the very first element, right-aligned, correct relative path (`../Portland Long.png` for this folder)
- [ ] Title follows `# Title: Strategic Briefing` pattern with an italicised date line beneath
- [ ] Horizontal rules (`---`) between every major section
- [ ] Key statistics presented in tables, not prose lists
- [ ] File named `<Topic>_Briefing.md` and located in `/docs/terminai/Briefings/`
- [ ] Closing italicised `*Sources: ...*` line lists all major references

## 2. Sources and Links

- [ ] ★ Every section containing statistics or factual claims ends with a `Sources:` line
- [ ] ★ Every link is an HTML anchor with `target="_blank"` — zero markdown-format links
- [ ] ★ Every URL was verified with WebFetch before inclusion — no unfetched links, however plausible
- [ ] Every statistic carries an inline citation; volatile figures (prices, market data) carry "as at [date]"
- [ ] Superseded-publication check done on government/regulatory citations
- [ ] Primary sources used for load-bearing claims; secondary commentary labelled as such

## 3. Factual QA

- [ ] ★ Numbers cross-checked: any figure appearing twice is identical both times; totals and percentages add up
- [ ] Units and currencies consistent (GBP unless stated; per-litre vs per-tonne not mixed silently)
- [ ] Dates current — nothing presented as "latest" that has a newer release (check publication cycles: DUKES, ONS, etc.)
- [ ] Where sources disagree, the disagreement is reported, not averaged away
- [ ] Estimates, forecasts, and opinions explicitly labelled as such — never presented as fact
- [ ] Uncertainty and data gaps stated somewhere visible, not buried

## 4. Fitness for Purpose

- [ ] Audience and purpose (agreed at the outline stage) actually served — re-read the brief before sign-off
- [ ] Executive summary readable standalone: a reader who stops there gets the correct headline picture
- [ ] If a "one page" version was requested: 80-line maximum, all items the user flagged "important/critical" included
- [ ] Actionable conclusions/recommendations present — a briefing that only describes is unfinished
- [ ] No orphan sections from the agreed outline (either written or consciously dropped with the user's agreement)
